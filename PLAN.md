# 카드뉴스 에디터 기획서

> 참조 이슈: [FLYLIKEB/ChaLog#187 - 홍보 인스타 계정 + 카드뉴스](https://github.com/FLYLIKEB/ChaLog/issues/187)

---

## 1. 프로젝트 개요

### 목적
글감(텍스트)을 입력하면 자동으로 카드뉴스 이미지를 생성하고, 각 페이지의 배경 이미지를 마음에 들 때까지 반복 수정할 수 있는 에디터 도구를 만든다.

### 핵심 가치
- **편집 가능한 결과물 산출**: 자동 생성 후에도 사용자가 직접 수정할 수 있어야 한다
- **적절한 배경 이미지 자동 포함**: 글의 맥락에 맞는 배경 이미지를 자동으로 찾아 적용
- **반복 수정 가능**: 마음에 드는 결과가 나올 때까지 배경 이미지를 계속 재생성할 수 있다

---

## 2. 핵심 기능

### 2-1. 글 → 카드뉴스 자동 변환
- 긴 글(블로그 포스트, SNS 글, 기사 등)을 입력
- AI가 핵심 내용을 분석해 카드뉴스 페이지별로 분할
- 각 페이지에 적합한 텍스트 레이아웃 자동 생성

### 2-2. 배경 이미지 자동 생성 / 재생성
- 각 페이지의 텍스트 내용에 어울리는 배경 이미지 자동 생성
- 페이지별로 **개별 재생성** 가능
- 재생성 횟수 제한 없음 (마음에 들 때까지)
- 이미지 스타일 옵션 제공 (미니멀, 컬러풀, 다크, 라이트 등)

### 2-3. 카드뉴스 편집
- 텍스트 위치 / 크기 / 색상 조정
- 배경 이미지 불투명도(오버레이) 조정
- 페이지 순서 변경 / 추가 / 삭제
- 폰트 선택

### 2-4. 내보내기
- PNG / JPG 이미지로 페이지별 저장
- 전체 페이지 ZIP 일괄 다운로드
- 인스타그램 최적 비율(1:1, 4:5, 9:16) 지원

---

## 3. 사용자 플로우

```
1. 글 입력
   └── 텍스트 직접 입력 또는 붙여넣기

2. 자동 생성
   └── AI가 글을 분석해 카드뉴스 페이지 구성
       ├── 핵심 문장 추출
       ├── 페이지 분할 (보통 5~10장)
       └── 각 페이지 배경 이미지 자동 생성

3. 편집
   └── 미리보기 화면에서 페이지별 확인
       ├── 마음에 안 드는 배경 → [재생성] 버튼 클릭
       ├── 텍스트 수정
       └── 레이아웃 조정

4. 내보내기
   └── 완성된 카드뉴스 이미지 다운로드
```

## 5. 기술 스택

### 5-1. 핵심 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| Frontend | Next.js (React) | 빠른 개발, 이미지 처리 용이 |
| 스타일링 | Tailwind CSS | 빠른 UI 구성 |
| 텍스트 분석 | Claude API (`claude-sonnet-4-6`) | 글 분석 및 카드뉴스 페이지 구성 |
| **결과물 포맷** | **pencil.dev (`.pen`)** | JSON 기반, AI가 생성·수정 가능, VS Code 등 코드 에디터에서 직접 편집 가능 |
| 배경 이미지 | **Unsplash API** (키워드 검색) | 적절한 이미지 자동 탐색, 무료 고품질 |
| 이미지 생성 (대안) | Google Gemini Image API / Replicate / FAL.ai | Unsplash에서 못 찾을 경우 AI 생성 |
| 내보내기 | `.pen` → PNG (pencil.dev 내장) | 편집 완료 후 최종 PNG 추출 |

### 5-2. 결과물 포맷 전략 — pencil.dev `.pen` 파일

@devdesign.kr 원문 인사이트:
> "png로 뽑는 경우 수정사항마다 AI에게 재요청해야 하는 불편함이 있고, Figma MCP는 별도 설치 및 횟수 제한이 있어 잘 되지 않았다. 해결책으로 **pencil.dev**를 사용했다."

**왜 `.pen` 포맷인가:**
- JSON 기반 → AI(Claude)가 파일을 직접 생성하고 읽을 수 있음
- VS Code, Cursor, Antigravity 등 코드 에디터에서 바로 열기 + 편집 가능
- PNG로 바로 굽지 않아서 사람이 최종 터치할 여지 항상 존재

```
Claude가 .pen 파일 생성 (JSON)
    ↓
코드 에디터(VS Code 등)에서 열어 직접 편집
    ↓
최종 확인 후 PNG로 내보내기
```

### 5-3. 배경 이미지 전략 — Unsplash API

- 각 페이지의 키워드로 Unsplash 검색 → 적절한 이미지 자동 선택
- 마음에 안 들면 같은 키워드로 다른 이미지 재탐색 (재생성 아닌 재검색)
- Unsplash에서 못 찾는 경우 Gemini / Replicate / FAL.ai로 AI 생성 fallback

| 서비스 | 방식 | 응답 형식 | 비고 |
|--------|------|----------|------|
| **Unsplash** | 키워드 검색 | URL | 무료, 고품질 사진 |
| **Google Gemini** | AI 생성 | base64 PNG | Google AI Studio API 키 |
| **Replicate** | AI 생성 | URL (비동기) | SDXL, Flux 등 다양한 모델 |
| **FAL.ai** | AI 생성 | URL (동기/비동기) | Flux 계열 강점 |

## 6. 워크플로우 아키텍처

@devdesign.kr + @hyojin.rgb 두 사례를 통합해 설계한다.

### 6-1. 에이전트 마인드셋 — "대표가 인턴에게 일을 시킨다"

> (@hyojin.rgb) "매거진 하나를 만들기 위해 필요한 인력: 1) 에디터 2) 콘텐츠 마케터 3) 사진가. 각각 인턴이라고 상상해서 작업한다."

각 에이전트는 독립된 페르소나를 가지고, **각자의 전략서(Strategy MD) + SOP**를 기반으로 작업한다.

### 6-2. 에이전트별 역할 및 산출물

| 에이전트 | 페르소나 | 입력 | 출력 |
|---------|---------|------|------|
| 오케스트레이터 | 대표 | 원본 글 | 전체 파이프라인 실행 |
| 편집장 | 에디터 인턴 | 원본 글 | 카드뉴스 구성안 JSON |
| 카피라이터 | 콘텐츠 마케터 인턴 | 구성안 | 페이지별 헤드라인 + 서브텍스트 |
| 아트디렉터 | 사진가 인턴 | 구성안 | Unsplash 키워드 + 이미지 선택 |
| 퍼블리셔 | - | 위 모든 결과 | `.pen` 파일 생성 |

### 6-3. 멀티 에이전트 파이프라인

```
오케스트레이터 (Entry Point / Command)
    │
    ├── 1단계: 편집장 에이전트
    │       └── 글 분석 → 카드뉴스 구성안 (페이지 수, 흐름, 핵심 메시지)
    │
    ├── 2단계: 병렬 실행 ──────────────────────────────┐
    │       ├── 카피라이터 에이전트                     │
    │       │       └── 페이지별 헤드라인 + 서브텍스트  │
    │       └── 아트디렉터 에이전트                     │
    │               └── Unsplash 키워드 + 이미지 선택   │
    │                                                  ┘
    └── 3단계: 퍼블리셔 에이전트
            └── .pen 파일 생성 (JSON 조합 → pencil.dev 포맷)
```

### 6-4. 폴더 구조 기반 작업 공간

> (@hyojin.rgb) "프로젝트에 폴더 구조를 만들어서 작업 공간을 로컬에서 알아서 찾으라고 했다. 폴더 구조 분석해서 알아서 잘 찾는다."

```
card-news-project/
├── input/              # 원본 글 텍스트
├── references/         # 원하는 디자인 레퍼런스 스크린샷
├── strategy/           # 에이전트별 전략 MD 파일
│   ├── editor.md       # 편집장 전략서 + SOP
│   ├── copywriter.md   # 카피라이터 전략서 + SOP
│   └── artdirector.md  # 아트디렉터 전략서 + SOP
├── output/             # 생성된 .pen 파일
└── export/             # 최종 PNG 이미지
```

### 6-5. 전략서 + SOP 구조

> (@hyojin.rgb) "인턴은 SOP가 없으면 일하기 어려우니, 클로드에게 SOP를 만들어서 일하라고 한다."

각 에이전트의 전략 MD 파일에 포함할 내용:
- **역할 정의**: 이 에이전트가 맡은 업무 범위
- **작업 맥락**: 전체 카드뉴스 제작에서의 위치
- **SOP (표준 작업 절차)**: 단계별 처리 방법
- **품질 기준**: 결과물이 충족해야 할 조건

### 6-6. 레퍼런스 엔지니어링

> (@devdesign.kr) "원하는 카드뉴스 디자인 레퍼런스를 디렉토리에 넣고, 레퍼런스별 템플릿을 만드는 SKILL을 등록하면 디자인이 잘 뽑힌다."

- `/references/` 디렉토리에 원하는 카드뉴스 디자인 스크린샷 저장
- 아트디렉터 에이전트가 레퍼런스 분석 → 해당 스타일 기반으로 `.pen` 템플릿 생성
- 레퍼런스가 많아질수록 결과물 품질 향상

### 6-7. 글 분석 프롬프트 (편집장 에이전트)
- 입력: 원본 글
- 출력: 카드뉴스 구성 JSON
  ```json
  {
    "title": "전체 제목",
    "pages": [
      {
        "page": 1,
        "headline": "핵심 문장 (짧게)",
        "subtext": "부연 설명 (선택)",
        "imageKeyword": "Unsplash 검색 키워드 (영문)"
      }
    ]
  }
  ```

## 7. MVP 범위

### 포함
- [ ] 텍스트 입력 → 카드뉴스 페이지 자동 분할
- [ ] 페이지별 배경 이미지 자동 생성
- [ ] 배경 이미지 개별 재생성
- [ ] 기본 텍스트 편집 (위치, 색상)
- [ ] PNG 이미지 저장

### 제외 (추후 기능)
- 인스타그램 직접 게시 연동
- 템플릿 저장/불러오기
- 팀 협업 기능
- 동영상 릴스 변환

## 8. 성공 기준

- 글 입력 후 카드뉴스 초안 생성까지 **30초 이내**
- 배경 이미지 재생성 **5초 이내**
- 생성된 카드뉴스가 인스타그램에 바로 올릴 수 있는 퀄리티
- 비개발자도 직관적으로 사용 가능한 UX
