# /handoff — 핸드오프 메모 업데이트

현재 프로젝트 상태를 분석하여 핸드오프 메모(MEMORY.md)를 최신 상태로 업데이트합니다.
다음 세션의 Claude가 프로젝트를 바로 이해하고 작업을 이어갈 수 있도록 합니다.

## 입력

- `$ARGUMENTS` : (선택) 추가로 기록할 내용이나 특별 메모. 없으면 자동 분석.

## 실행 절차

1. **현재 상태 파악**
   - `git log --oneline -20`으로 최근 커밋 확인
   - `git status`로 미커밋 변경사항 확인
   - `git diff --stat`으로 변경된 파일 범위 파악
   - 주요 파일들(page.tsx, 컴포넌트, API routes, types) 빠르게 스캔

2. **MEMORY.md 업데이트 항목**
   - **현재 상태 날짜** 갱신
   - **완료된 기능** 목록 — 새로 완료된 것 추가, 변경된 것 수정
   - **핵심 아키텍처** — 구조 변경 시 업데이트
   - **주요 파일** 테이블 — 새 파일 추가, 삭제된 파일 제거
   - **진행 중 / 미완료 작업** — 현재 작업 중인 것, 다음에 해야 할 것
   - **발견된 버그 / 이슈** — 알려진 문제점
   - **사용자가 $ARGUMENTS로 전달한 추가 메모**

3. **MEMORY.md 파일 수정**
   - 경로: `~/.claude/projects/-Users-jwp-Documents-programming-card-news-edditor/memory/MEMORY.md`
   - 기존 구조를 유지하면서 내용만 업데이트
   - 200줄 이내로 유지 (truncation 방지)
   - 불필요한 과거 정보는 정리

4. **결과 보고**

## 출력 형식

```
✅ 핸드오프 메모 업데이트 완료

📝 변경 요약:
- [업데이트한 섹션 목록]
- [추가된 주요 항목]
- [제거/정리된 항목]

다음 세션에서 이 메모를 자동으로 참조합니다.
```

## 주의사항

- MEMORY.md는 200줄 제한이 있으므로 간결하게 유지
- 세션별 임시 정보는 기록하지 않음
- 검증된 사실만 기록 (추측 금지)
- 코드를 직접 읽어서 확인한 후 기록
